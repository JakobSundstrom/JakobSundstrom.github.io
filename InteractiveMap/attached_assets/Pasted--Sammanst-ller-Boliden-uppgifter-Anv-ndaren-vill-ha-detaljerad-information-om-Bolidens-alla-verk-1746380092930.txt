## Interactive Map-Quiz Game Concept

**1. Overview & Purpose**
An engaging, web-based educational quiz that:

* **Visualizes** all Boliden operations as clickable dots on a Europe-focused map.
* **Educates** players on each site’s history, location, and products through interactive questions.
* **Tracks** scores, times and provides instant feedback to reinforce learning.

**2. Main Screens & Flow**

1. **Title Screen**

   * Boliden branding, “Start Quiz” button, difficulty level selector (e.g. Basic vs. Expert).
2. **Map View**

   * Stylized map of Northern Europe with dots for each mine & smelter.
   * Hover tooltip shows site name; click to “select.”
3. **Question Panel**

   * After selecting a dot, panel slides in with:

     * **Site Name & Image** (e.g. photo or icon)
     * **Multiple-Choice Question**, e.g.:

       * “When did Garpenberg first begin operations?”
       * “Which metal is NOT a primary output of Aitik?”
     * **4 Answer Options** (one correct, three distractors).
   * Immediate feedback: ✔️ or ❌ + brief fact.
4. **Progress Tracker**

   * Shows number of questions answered, current score, and elapsed time.
5. **Results Screen**

   * Summary: total correct, areas for improvement, option to retry or review all facts.

**3. Question Types & Examples**

* **Location Identification**:

  * “Click on the mine that produces nickel and copper concentrates.”
* **Historical Dates**:

  * “In what year did the Rönnskär smelter commence operations?”
* **Product Matching**:

  * “Match the Harjavalta smelter to its main products.”
* **True/False Facts**:

  * “True or False: The Odda smelter produces sulphuric acid.”

**4. Visual & UI Elements**

* **Map Interface**:

  * Clear, high-contrast dots with hover highlighting.
  * Responsive layout for desktop/tablet.
* **Panels & Modals**:

  * Slide-in panels for questions, with icons/photos of each site.
  * Clean typography, Boliden’s color palette (greens/blues).
* **Feedback Animations**:

  * Subtle confetti or shake effect on correct/incorrect answers.

**5. Backend & Data Structure**

* **JSON Data** for each site, e.g.:

  ```json
  {
    "id": "garpenberg",
    "name": "Garpenberg Mine",
    "coords": [60.3083, 16.1929],
    "start": "375 BC",
    "products": ["Zinc", "Lead", "Silver", "Gold"],
    "questions": [
      {
        "type": "date",
        "question": "When was Garpenberg first in operation?",
        "correct": "375 BC",
        "options": ["1553", "1972", "375 BC", "1957"]
      },
      {
        "type": "products",
        "question": "Which of these is NOT a main product of Garpenberg?",
        "correct": "Copper",
        "options": ["Zinc", "Lead", "Silver", "Copper"]
      }
    ]
  }
  ```
* **Scoring Logic**: simple +1 per correct, −0.5 for wrong, bonus for speed.

**6. Extension Ideas**

* **“Expert Mode”** with deeper history (e.g. medieval state enterprise dates).
* **Leaderboard** so colleagues can compete.
* **Mobile App** version using native maps and touch interactions.
